Les cartes réseau

Une carte réseau est un composant matériel qui permet à un ordinateur de se connecter à un réseau informatique via Ethernet ou Wi-Fi. Elle gère les communications entre l'ordinateur et le réseau

Switch

Un switch est un équipement réseau qui permet de connecter plusieurs périphériques entre eux au sein d'un réseau local. Il analyse les adresses MAC des périphériques connectés pour acheminer les données uniquement vers le périphérique destinataire, ce qui réduit la congestion du réseau par rapport à un hub

Hub

Un hub est un équipement réseau qui permet de connecter plusieurs périphériques entre eux au sein d'un réseau local. Il fonctionne en diffusant les données à tous les périphériques connectés, ce qui peut entraîner une congestion du réseau. Les hubs sont donc moins efficaces que les switches pour transférer les données. Les hubs sont généralement moins utilisés de nos jours car les switches offrent des performances plus élevées

2 - Le Modele OSI

Le modèle OSI (Open Systems Interconnection) est un modèle de référence qui décrit comment les différents composants d'un réseau informatique communiquent entre eux. Il est composé de 7 couches, chacune ayant un rôle spécifique dans la communication des données :

  • La couche physique : elle définit les caractéristiques électriques, mécaniques et fonctionnelles des connexions physiques du réseau.
  • La couche liaison de données : elle permet d'assurer une communication fiable entre les nœuds du réseau en détectant et corrigeant les erreurs de transmission.
  • La couche réseau : elle permet de gérer la transmission des données à travers le réseau en utilisant des protocoles de routage.
  • La couche transport : elle permet d'assurer une communication fiable et transparente entre les applications en segmentant les données et en contrôlant leur transmission.
  • La couche session : elle permet d'ouvrir, de maintenir et de fermer des sessions de communication entre les applications.
  • La couche présentation : elle permet de gérer la représentation des données pour les rendre compréhensibles par les applications.
  • La couche application : elle permet d'interagir avec les applications utilisateurs.

-En utilisant le modèle OSI, les fabricants de matériel et de logiciels peuvent concevoir des produits compatibles avec les différents composants d'un réseau informatique, assurant ainsi une interopérabilité entre les équipements de différents fournisseurs.

3 - Le Modèle TCP/IP

Le modèle TCP/IP (Transmission Control Protocol/Internet Protocol) est un modèle de communication qui décrit comment les ordinateurs communiquent entre eux sur Internet. Il est composé de 4 couches, chacune ayant un rôle spécifique dans la transmission des données :

  • La couche réseau : elle est responsable de la transmission des paquets de données sur le réseau et utilise le protocole Internet (IP) pour acheminer les paquets de données vers leur destination.
  • La couche transport : elle permet d'assurer une communication fiable entre les applications en utilisant le protocole de transport TCP ou le protocole d'datagramme utilisateur (UDP).
  • elle fournit des services de communication pour les applications, telles que le protocole HTTP pour les sites web, le protocole SMTP pour les emails, le protocole FTP pour le transfert de fichiers, etc.
  • La couche interface réseau : elle est responsable de la transmission des données entre les ordinateurs et les réseaux physiques.

-Le modèle TCP/IP est largement utilisé dans les réseaux informatiques, en particulier sur Internet. Les différents équipements réseau (routeurs, commutateurs, etc.) et les ordinateurs doivent être configurés avec des adresses IP uniques pour communiquer sur le réseau.

Adressage

Les communications dans un réseau obéissent à des règles

  • Adressage qui permet d'identifier de manière unique les deux unités en communication
  • Architecture qui définit les rôles endossés par les deux unités
  • Protocoles qui assurent l'échange des données

L'adressage IP

est le numéro qui identifie chaque ordinateur dans les réseaux qu’il appartient. Il y’a deux mode d’adressage IPV4 et IPV6 :

  • ADRESSE IPV4: Chaque paquet contient adresse source sur (32 bits) et une adresse destinataire sur (32bits)
    Donc IPV 4 == 4 octets ==32 bits
  • ADRESSE IPV6 : Les adresses sont codées sur 128 bits. La représentation textuelle se fait en découpant le mot de 128 bits de l'adresse en 8 mots de 16 bits séparés par le caractère «:», chacun d'eux est en hexadécimal

Adresse IPV4

  • Chaque adresse IPV4 contient deux parties :
    • Une première partie des bits de valeur supérieur : Adresse réseau
    • Une deuxième parties des bits : Adresse hôte
  • On reconnait qu’un réseau est un {hôte} quand la valeur des bits pour la partie réseau est la même
  • Le nombre des bits contenus dans la partie hôtes détermine le nombre des machines possible sur ce réseau

Les différents types d’adresse IPV4

  • Adresse réseau : tous les bits de la partie hôtes=0 exemple:( 10.0.0.0) ,c'est-à-dire tous les hôtes(machines) de ce réseau ont la même partie réseau
  • Adresse diffusion : c'est une adresse spéciale qui est utilisée pour envoyer des données à tous les périphériques sur un réseau. Elle est généralement utilisée pour envoyer des messages de diffusion ou pour détecter des périphériques sur le réseau.
  • Adresse hôte : est la partie unique de l'adresse IP d'un périphérique sur un réseau, utilisée pour l'acheminement des données vers ce périphérique spécifique. Elle est composée des derniers octets de l'adresse IPv4.
  • Le masque de réseau et de sous réseaux : Tous les bits de la partie réseaux = 1
